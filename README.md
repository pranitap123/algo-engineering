# Algo Engineering

> Building algorithmic thinking through deliberate practice, pattern recognition, and engineering-first problem solving.

---

## 🎯 About

**Algo Engineering** documents my journey toward mastering Data Structures & Algorithms for backend software engineering interviews.

This repository is not a collection of copied solutions or a count of solved problems.

Its purpose is to build:

- Logical thinking
- Problem-solving ability
- Pattern recognition
- Algorithmic intuition
- Clean coding habits
- Interview communication skills

Every solution is accompanied by explanations, complexity analysis, mistakes, and lessons learned.

---

## 🚀 Objectives

- Develop strong DSA fundamentals
- Learn algorithms by understanding, not memorization
- Build reusable problem-solving patterns
- Prepare for backend software engineering interviews
- Practice consistently through daily problem solving
- Maintain concise revision notes and templates

---

## 💻 Primary Language

**TypeScript**

I chose TypeScript to focus on algorithmic thinking while reinforcing the language I use for backend development.

## 📖 Learning Philosophy

I don't want to memorize solutions.

I want to understand:

- Why an algorithm works
- When a pattern should be applied
- How to derive solutions independently
- How to communicate ideas during interviews

The goal is to become an engineer who can solve unfamiliar problems, not someone who recognizes only previously seen questions.

## 📝 Daily Practice Workflow

For every problem:

1. Understand the problem
2. Think independently
3. Explain the brute-force approach
4. Optimize step by step
5. Analyze complexity
6. Write clean code
7. Record mistakes
8. Summarize key learnings
